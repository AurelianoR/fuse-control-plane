Methodology

How Fuse works

Five steps. Zero network disruption.

1
Observe
Polls cloud IAM APIs out-of-band to inventory every active brokered session across AWS, Azure, and GCP
2
Score
Automatically risk-scores each token by expiry, scope permissiveness, and API usage vs. quota
3
Alert
Surfaces Critical and High risk sessions instantly in the dashboard with pulsing visual indicators
4
Revoke
On operator command, calls the provider's IAM control plane to invalidate the session at root in under 1 second
5
Log
Writes an RFC3339-timestamped immutable audit entry with full context — ready for ISMS auditors
Capabilities

Built for cloud SecOps teams

Every feature designed around the real workflows of cloud security and governance teams.

🔍
Multi-Cloud Token Visibility
Single inventory of every brokered IAM session across AWS STS, Azure Entra ID, and GCP Cloud IAM. No console switching.
📊
Live API Usage Monitoring
Per-session gradient progress bars showing API calls consumed vs. session quota. Auto-refreshes every 10 seconds from the Go backend.
⚡
One-Click Revocation
Click Revoke Access → confirm → done. The Go backend calls the cloud provider IAM API to invalidate the session at the root level.
📋
Immutable Audit Trail
Every revocation is permanently logged with RFC3339 timestamps, vendor, provider, scope, and actor. Survives reboots. Ready for ISMS auditors.
🔴
Automated Risk Scoring
Low / Medium / High / Critical. Static keys, over-permissive roles (roles/editor), and quota-exceeded sessions are flagged automatically.
📐
Compliance Posture Meters
Live framework coverage bars for ISO/IEC 27001, SOC 2 Type II, NIST CSF 2.0, and CIS Cloud Benchmark — exportable as evidence.
🛡️
Out-of-Band Architecture
Fuse never proxies production traffic. If it goes down, your cloud workloads continue unaffected. Zero single point of failure introduced.
🔄
Graceful Offline Mode
If the Go API is unreachable, the dashboard falls back to mock data with a clear sidebar indicator. Operators always know their data source.
🔍
Advanced Filtering
Filter the sessions table by cloud provider (Azure, AWS, GCP) or show only Critical-risk sessions. Filter state persists across auto-refreshes.
REST API

Simple, clean endpoints

The Go backend exposes 5 endpoints. All return JSON. All support CORS. Built to integrate with your SIEM and alerting stack.

GET
/api/health
Kubernetes liveness probe. Returns 200 when the API is ready to serve requests.
{ "status": "ok", "service": "fuse-control-plane" }
GET
/api/tokens
Returns all active non-revoked brokered sessions with live usage metrics.
[{ "id": "tok_az_1", "vendor": "Datadog",
"provider": "azure", "risk_level": "Low",
"token_usage": 657, "usage_limit": 5000 }]
POST
/api/tokens/revoke
Revokes a session by ID. Writes an immutable audit entry. Returns success or 404 if not found.
curl -X POST /api/tokens/revoke \
-d '{"token_id":"tok_gcp_3"}'

→ { "status": "success" }
GET
/api/metrics
Aggregated snapshot for KPI cards — total tokens, per-provider counts, critical session count.
{ "total_tokens": 5,
"by_provider": {"azure":2,"aws":2,"gcp":1},
"critical_count": 1 }
GET
/api/audit
Returns the full immutable revocation log. Each entry is a timestamped RFC3339 string with full context for compliance evidence.
{ "total": 1, "entries": [
"[2026-06-24T15:01:42+02:00] REVOKED | ID: tok_gcp_3 | Vendor: External Dev Agency | Provider: gcp | Scope: roles/editor (Over-Permissive)"
]}

Technology

Cloud-native from the ground up

Every technology choice made for performance, security, and operational simplicity.

🐹
Go 1.21
Zero-dependency static binary. Sub-millisecond latency. The industry standard for cloud infrastructure tooling. ~12MB Docker image.
🐋
Docker (Multi-stage)
Multi-stage build compiles the binary in Go 1.21, copies only the binary into Alpine 3.19. Non-root user. CA certs for cloud API calls.
🟣
Terraform 1.5+
Declarative, state-managed provisioning of the entire Azure infrastructure. Reproducible environments. One command to provision, one to destroy.
☸️
Azure AKS
Managed Kubernetes with System-Assigned Managed Identity. No credentials to manage. Native integration with ACR and Log Analytics.
🌐
Vanilla HTML/CSS/JS
No build step. No framework CVE surface. Instant load. The dashboard works from a file:// URL or served by any static server.
📊
Azure Log Analytics
90-day audit log retention for all cluster events. KQL queryable. ISMS-compliant. Integrated at AKS level — zero additional configuration.
Quick Start

Running in 30 seconds locally

One Docker Compose command gives you the full stack. Terraform provisions production in ~8 minutes.

🐋 Local — Docker Compose
git clone github.com/YOUR_ORG/fuse-control-plane
cd fuse-control-plane

docker compose up --build

# Dashboard → http://localhost:3000
# API → http://localhost:8080
🐹 Local — Go directly
cd backend && go run main.go
# 🚀 Fuse API listening on :8080

open frontend/index.html

# Verify:
curl localhost:8080/api/health
curl localhost:8080/api/tokens
🟣 Production — Terraform + AKS
az login
cd infrastructure
terraform init
terraform apply

# Build & push to ACR:
ACR=$(terraform output -raw acr_login_server)
docker build -t $ACR/fuse-api:latest ./backend
docker push $ACR/fuse-api:latest
☸️ Deploy to AKS
eval "$(terraform output -raw aks_get_credentials_cmd)"

kubectl create deployment fuse-api \
--image=$ACR/fuse-api:latest \
--replicas=2

kubectl expose deployment fuse-api \
--type=LoadBalancer --port=80 \
--target-port=8080
