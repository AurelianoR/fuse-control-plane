Methodology

Three tiers, value on day one

The lower tiers need nothing from any vendor. Binding is reserved for the connections worth the effort.

1
Connect
Add a read-only connector (Azure/Entra, GitHub App, demo) under a company. No vendor change required.
2
Discover
Fuse inventories every grant, app and token — scope, age, last used, by whom, consent chain, publisher.
3
Score & flag
Each connection gets risk signals (write-perms, unverified publisher, dormant, unbound) and compliance checks.
4
Policy
Set short lifetimes, minimize scope, or revoke — per token or per company. Cuts the breach window from months to minutes.
5
Bind (DPoP)
For high-value connections, the gateway verifies a fresh sender-bound proof on every call — a stolen token alone fails.
Capabilities

Built for token governance

Everything a security team needs to see, constrain and lock down third-party access — in one console.

🪪
Token Monitor
A live grant inventory of every OAuth grant, app install and PAT. Click any token for the full deep-dive: last used, by whom, consent chain, publisher, permissions, and event history.
🔒
DPoP Sender-Binding
The differentiator. The Gateway verifies a fresh proof on every call — four checks (token, key match, request, freshness). A stolen token without the private key fails. Standards-based RFC 9449.
🛡️
Policy per Token & Company
Set token lifetime, scope and DPoP binding on each gateway token, or apply a group policy to a whole company in one click. Short lifetimes cut the breach window from months to minutes.
🏢
Company-Scoped Console
Organize connectors under companies and switch between them from the sidebar. The Dashboard and Connectors scope to the selected company; "All companies" shows each one side by side.
⚡
Revoke — or Redirect
Cut a connection's access from Fuse where possible. For tokens Fuse can't cut itself, the detail screen links straight to the provider's own revoke page (Entra admin, GitHub installations).
✅
Compliance Findings
Negative status only — the concrete gaps that need action: write/admin scope, unverified publisher, dormant, user-consented, unbound bearer. Each finding links to the affected connections.
🔌
Real Connectors
Not mocked. The Azure/Entra connector enumerates real service principals & OAuth grants via Microsoft Graph; the GitHub App connector mints real installation tokens and lists org apps.
📋
Audit Log + Export
Every connector sync, policy change, revocation, gateway allow/block and breach attempt is logged newest-first — and exportable as a timestamped report for auditors.
🌐
Out-of-Band by Default
For platforms that verify the proof themselves, Fuse is never in the request path — it's the management layer. Inline gateway mode is the rare fallback, your choice per connection.
JSON API

Same-origin, one FastAPI app

The console is served by Fuse on :8000 and calls its own API. Below are the endpoints behind each view.

GET
/api/grants
The Token Monitor inventory — live connector apps + the seeded grant store, grouped by company. Filter by source, platform, risk, type, search.
{ "sources": [{"id":"co-…","name":"Acme Corp"}],
"rows": [{ "id":"app-…", "vendor":"PipelineCRM",
"platform":"salesforce", "consent":"Admin",
"last_used_ago":"5m ago", "risk_signals":[…] }] }
GET
/api/grant/{id}
The full-screen detail: identity, activity (last used / by whom), permissions, a compliance checklist, event history, and provider-revoke links.
{ "vendor":"DocuSign", "risk_signals":[…],
"compliance": { "compliant": false, "failed": 3,
"checks": [{"label":"Sender-bound (DPoP)","ok":false}…] },
"provider_revoke": {"provider":"Microsoft Entra","url":"…"} }
GET
/api/policy/overview
Companies → connections with current policy, for the Policy view. Pairs with POST /api/apps/{id}/policy (per token) and POST /api/policy/bulk (per company).
{ "companies": [{ "name":"Acme Corp", "kind":"company",
"connections": [{ "id":"app-…", "governable":true,
"policy": {"lifetime_seconds":300,"binding_required":true} }] }] }
GET
/api/compliance/findings
Negative status only — concrete risk findings across the inventory, ranked by severity. Drives the Compliance view and the alert badge.
{ "total": 7, "by_severity": {"high":3,"medium":2,"low":2},
"findings": [{ "severity":"high", "signal":"write-permissions",
"title":"18 hold write/export/admin permissions" }] }
POST
/api/tokens/revoke
Cut a connection's access. Real Azure/GitHub connectors revoke at source; otherwise the detail screen offers a provider-redirect. Writes an audit entry.
curl -X POST /api/tokens/revoke \
-d '{"token_id":"app-b4343ef1"}'

→ { "status":"success", "message":"PipelineCRM: access cut" }
GET
/gateway/contacts
The inline DPoP gateway. Verifies the four binding checks on every request and forwards only verified calls. Driven from the Gateway view's legit + attacker runs.
Authorization: DPoP <token>
DPoP: <fresh signed proof>

→ 200 forwarded · or 401 "proof key does not match"

Technology

Real crypto, no hand-rolled auth

Standards-based primitives so no one can say the binding was invented here.

🐍
Python · FastAPI
Three async services on uvicorn: the Fuse console & token authority (:8000), a mock company platform (:8010), and a vendor that holds keys and signs proofs (:8020).
🔑
DPoP · ES256 · private_key_jwt
jwcrypto + PyJWT. RFC 9449 sender-bound proofs with a cnf.jkt thumbprint, RFC 7523 client assertions. The four binding checks are real and verifiable.
🗄️
SQLAlchemy · SQLite
The grant inventory store, seeded with realistic demo tenants across two collection runs (so you see new / removed / permission-added deltas and dormancy).
☁️
Microsoft Graph · GitHub App
Real connectors via MSAL + httpx: Entra service principals & oauth2PermissionGrants; GitHub App installation tokens and org app inventory.
🌐
Vanilla HTML/CSS/JS
One self-contained SPA, no build step, no framework CVE surface. Served same-origin by Fuse so the browser only ever talks to one host.
🧩
Connector framework
Each source normalizes into Apps + Tokens behind one interface — demo, GitHub, Azure today; Salesforce / Google next. Connecting is always manual & reversible.
Quick Start

Running locally in a minute

One script starts all three services and seeds the demo grant inventory. Everything is same-origin on :8000.

① Install
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
② Run all services
./run.sh

# Company API → :8010 Vendor → :8020
# (grant store seeded automatically)
